## Recent Releases

<Update label="2024-10-15" description="v1.2.0" tags={["feature", "improvement"]}>

## New Features

- Added real-time dashboard at `https://dashboard.example.com` for monitoring API usage.
- Introduced API v2 endpoints with enhanced performance.

## Improvements

- Reduced latency by `>50%` on core requests to `https://api.example.com/v2/users`.
- Updated webhook validation for better security.

## Bug Fixes

- Fixed intermittent failures in token refresh flows.
- Resolved UI glitches on mobile devices.

</Update>

<Update label="2024-09-20" description="v1.1.0" tags={["feature", "bugfix"]}>

## New Features

- Multi-factor authentication (MFA) support for `https://auth.example.com`.
- Webhook events for user actions.

## Bug Fixes

- Patched rate limiting edge cases causing `429` errors.
- Corrected pagination in list endpoints.

<Update label="2024-08-10" description="v1.0.0" tags={["feature", "breaking"]}>

## New Features

- Core API launch with user management at `https://api.example.com/users`.
- Dashboard integration for analytics.

## Breaking Changes

- Migrated from v0.x auth headers to `Authorization: Bearer {YOUR_TOKEN}`.

</Update>

<Expandable title="Migration Guide" default-open="false">

If upgrading from v1.0.0, update your requests:

````javascript
// Old
fetch('https://api.example.com/users', {
  headers: { 'X-API-Key': 'YOUR_API_KEY' }
});

// New
fetch('https://api.example.com/v2/users', {
  headers: { 'Authorization': `Bearer ${YOUR_TOKEN}` }
});
````

</Expandable>
