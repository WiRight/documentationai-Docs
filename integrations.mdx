## Setting Up Third-Party Connections

Begin by navigating to the Integrations section in your IDSMK dashboard at `https://dashboard.idsmk.ru/integrations`. Enable the desired connection and follow the prompts.

<Callout kind="info">
  You need admin access to configure integrations. Ensure your account has the necessary permissions.
</Callout>

Popular no-code options include Zapier and Make.com. For example:

- **Zapier**: Search for IDSMK in the app directory and authenticate with your `API_KEY`.
- **Slack**: Use webhooks to send alerts directly to channels.

## Configuring Webhooks

Webhooks push real-time events from IDSMK to your endpoint. Follow these steps to set one up.

<Steps>
  <Step title="Create Endpoint" icon="server">
    Set up a public HTTPS endpoint on your server to receive POST requests.
  </Step>
  <Step title="Add Webhook in Dashboard" icon="settings">
    Go to Integrations > Webhooks > New Webhook. Enter your endpoint URL like `https://your-webhook-url.com/idsmk`.
  </Step>
  <Step title="Select Events" icon="bell">
    Choose events such as `user.created` or `payment.failed`.
  </Step>
  <Step title="Test and Save" icon="check-circle">
    Send a test event and verify receipt.
  </Step>
</Steps>

Here's an example payload:

<CodeGroup tabs="JSON">
  ```json
  {
    "event": "user.created",
    "data": {
      "id": "usr_123",
      "email": "user@example.com",
      "timestamp": "2024-01-15T10:30:00Z"
    }
  }
  ```
</CodeGroup>

## API Integration Basics

Use the IDSMK REST API at `https://api.idsmk.ru/v1` for programmatic access. Authenticate requests with a Bearer token.

<ParamField header="Authorization" param-type="string" required="true">
  Bearer `{YOUR_API_KEY}`
</ParamField>

<ParamField query="event_type" param-type="string" required="false">
  Filter events, e.g., `user.created`.
</ParamField>

<Tabs>
  <Tab title="JavaScript" icon="code">
    ```javascript
    const response = await fetch('https://api.idsmk.ru/v1/events', {
      method: 'GET',
      headers: {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
      }
    });
    const data = await response.json();
    console.log(data);
    ```
  </Tab>
  <Tab title="Python" icon="python">
    ```python
    import requests

    headers = {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
    }
    response = requests.get('https://api.idsmk.ru/v1/events', headers=headers)
    data = response.json()
    print(data)
    ```
  </Tab>
</Tabs>

For full request/response examples:

<Request tabs="JavaScript,cURL">
  ````javascript
  const response = await fetch('https://api.idsmk.ru/v1/users', {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY'
    },
    body: JSON.stringify({
      email: 'newuser@example.com'
    })
  });
  ````

  ````bash
  curl -X POST https://api.idsmk.ru/v1/users \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{"email": "newuser@example.com"}'
  ````
</Request>

<Response>
  ```json
  {
    "id": "usr_456",
    "email": "newuser@example.com",
    "status": "active"
  }
  ```
</Response>

## Troubleshooting Common Issues

<ExpandableGroup>
  <Expandable title="Webhook Not Receiving Events" default-open="true">
    Check your endpoint logs for 200 OK responses. Verify the URL uses HTTPS and is publicly accessible. Test with tools like ngrok for local development.
  </Expandable>
  <Expandable title="401 Unauthorized Errors">
    Ensure your `API_KEY` is valid and not expired. Regenerate if needed from the dashboard.
  </Expandable>
  <Expandable title="Timeout Issues">
    IDSMK timeouts after 10 seconds. Optimize your endpoint response time.
  </Expandable>
</ExpandableGroup>

<Callout kind="tip">
  Monitor integration logs in the IDSMK dashboard for detailed error messages. Contact support at support@idsmk.ru for persistent issues.
</Callout>
